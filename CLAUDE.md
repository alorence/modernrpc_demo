## Project overview

This is a demo website for the **django-modern-rpc** library, which allows exposing RPC (Remote Procedure Call) procedures in a Django application. The project demonstrates various use cases of the library with working RPC procedure examples.

## Tech stack

- **Backend**: Django 6.0+ (Python 3.14+) with django-modern-rpc v2.0+
- **Frontend**: Alpine.js, Tailwind CSS 4, FlyOnUI
- **Dependency management**:
  - Python: `uv` (modern package manager)
  - JavaScript: `pnpm`
- **Production server**: Gunicorn with UvicornWorker (ASGI)
- **Deployment**: Multi-stage Dockerfile (Node + Python Alpine)

## Essential commands

### Initial setup
```bash
# Install Python and Node.js dependencies
uv sync
pnpm install
```

### Development
```bash
# Start all services in parallel (Django + CSS watcher + JS bundler)
uvx honcho start

# OR start services individually:
uv run python manage.py runserver  # Django server
pnpm run dev:css                   # Tailwind CSS watcher
pnpm run dev:js                    # Rollup in watch mode
```

### Production build
```bash
pnpm run build:prod  # Compile CSS (minified) and bundle JS
python manage.py collectstatic --no-input
```

### Development with a local version of modernrpc
```bash
# Install modernrpc in editable mode
uv pip install -e ../modernrpc

# OR edit pyproject.toml to uncomment:
# django-modern-rpc = { path = "../modernrpc", editable = true }
```

## Code architecture

### Directory structure
```
core/           # Main Django configuration
  settings.py   # Configuration with django-environ (.env support)
  urls.py       # Main URL routing
  asgi.py       # ASGI entry point with DjangoUvicornWorker

main/           # Main Django application
  rpc/          # RPC modules organized by namespace
    server.py   # RpcServer instance with Sentry error handler
    math.py     # "math" namespace: add, subtract, multiply, divide, power, sqrt
    errors.py   # "errors" namespace: error handling examples
    meta.py     # "utils" namespace: metadata and utilities
    http.py     # "http" namespace: HTTP headers/request manipulation
  views.py      # Main view (HomePageView)
  urls.py       # Routes: / (home), /rpc (sync), /async-rpc (async)
  templates/    # Django templates with auto-generated procedure documentation
  templatetags/ # Custom tags for RPC display

www/            # Frontend sources
  index.js      # Alpine.js entry point
  index.css     # Tailwind CSS entry point

dist/           # Compiled assets (generated by build)
```

### RPC system

The RPC server is configured in `main/rpc/server.py`:
- **Central instance**: `RpcServer(error_handler=main_error_handler)`
- **Namespaces**: Procedures are organized into namespaces (`math`, `errors`, `utils`, `http`)
- **Endpoints**:
  - `/rpc` → Synchronous view (`server.view`)
  - `/async-rpc` → Asynchronous view (`server.async_view`)
- **Error handling**: Exceptions are captured and sent to Sentry via `main_error_handler`

### RPC procedure organization

Each namespace is a separate module in `main/rpc/`:
- Create an `RpcNamespace()` instance
- Decorate functions with `@namespace.register_procedure(name="...")`
- Procedures are automatically exposed via the RPC server
- Documentation (docstrings) is used to generate the web interface

### Configuration and environment

The `core/settings.py` file uses **django-environ** to manage configuration:
- Environment variables in `.env` (optional)
- Key variables:
  - `DEBUG` (bool, default: False)
  - `DATABASE_URL` (default: in-memory sqlite)
  - `SECRET_KEY` (required in production)
  - `SENTRY_DSN` (optional, for monitoring)
  - `GTAG_PROPERTY_ID` (optional, Google Analytics)
  - `ENVIRONMENT` (str, default: "dev")
  - `ENABLE_DEBUG_LOGS` (bool, default: False)

### Frontend build pipeline

- **Tailwind CSS 4**: Compiles `www/index.css` → `dist/app.css`
  - Inline configuration in `www/index.css` (no tailwind.config.js)
  - Uses FlyOnUI for UI components
- **Rollup**: Bundles `www/index.js` → `dist/app.js`
  - Includes Alpine.js and related modules
  - Copies clipboard.js and helper-clipboard.js to dist/
- **Static files**: Django collects from `dist/` to `static/` with Whitenoise (compression + manifest)

### Deployment specifics

- **Conditional middleware**: `django_browser_reload` and `django_watchfiles` are automatically disabled if not installed (prod)
- **HTTPS in production**: Enabled if `ENVIRONMENT=prod` (HSTS, secure cookies, SSL redirect)
- **Docker**: Multi-stage build to optimize image size (frontend separated from Python runtime)
